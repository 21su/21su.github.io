나의 역할


공통역할 :


  1. 각각 생각해온 아이디어로 진행된 회의를 통해 프로젝트 기획 및 총괄

  2. 총괄 후 데이터베이스 정의, 설계 및 구현

  3. 프로젝트 시작하기 전 개개인의 역량에 맞춰 역할을 분배

  4. 프로젝트 시작하기 전 전체적인 자료조사


내 주요역할:


1. 지도 띄우기 및 전체 지도 관리

  - TMap API에서 제공해주는 앱키를 사용하여 ajax를 통해 TMap에 지도를 요청하여 화면에 출력함

  - 지도를 사용한 기능들을 조사

  - 마커 객체를 만들어 지도에 원하는 위치에 표시를 할 수 있게 만듦

  - 저장되어있는 원하는 장소를 바로 찾아주는 마이플레이스 기능과 늘 가던 길을 바로 찾아주는 루틴 기능을 구현

  - 총 4개의 탭으로 구성


2. 1 탭 지도 검색 기능

  - 1 탭은 검색 기능으로 AJAX를 통해 검색어를 TMap에 보내면 설정된 데이터에 따라 보고 있는 지도의 위치를 기준으로 6까지의 검색 결과를 객체로 받아와 HTML에 뿌려줌

  - 검색된 결과값은 지도에 마커로 표시되게 만들고 클릭 시 해당 마커의 이미지 색이 바뀌면서 경로 탐색을 위한 출발지와 도착지에 집어넣을 수 있는 버튼이 나오게 만듦

  - 로그인이 되어있으면 마이플레이스에 저장할 수 있게 버튼을 만들어 둠

  - 출발지와 도착지 버튼 클릭 시 2 탭으로 넘어가 출발지/도착지위치에 설정 함


3. 2탭 경로 탐색 기능

  - 2 탭에서는 맵 클릭 이벤트가 활성화 되어, 지도 클릭 시 클릭 위치에 출발지와 도착지를 차례대로 설정이 되게 만듦

  - 출발지와 도착지는 각각 출발지/도착지 마커를 이용하여 지도에 표시됨

  - 출발지/도착지 검색 창을 만들어 2탭에서도 검색이 가능하게 만들고, 검색 후에 나온 검색 결과를 통해 바로 출발지/도착지 설정을 가능하게 만듦

  - 출발지와 도착지가 모두 입력이 되면 AJAX로 출발지와 도착지 마커의 좌표값을 가져와 TMap으로 데이터를 보내고 그 후 TMap에서 받은 데이터를 기준으로 최적의 도보 경로를 탐색해 객체로 값을 이쪽으로 보내주면 그 객체를 통해 경로를 지도에 뿌린다.

  - 로그인이 되어있으면 경로 탐색이 완료된 후 그 경로 데이터를 루틴에 저장할 수 있게 버튼을 만들어둠


4. 3, 4 탭 조정

  - 팀원이 만든 3 탭과 4 탭을 기존에 만든 1, 2 탭과 호환되게 조정하고, 어색함이 없게 스타일을 조정함

  - 모든 탭을 하나의 창에서 이루어질 수 있게 코드를 합침


5 마이플레이스와 루틴 기능

  - 마이플레이스/루틴 테이블과 멤버 테이블을 연동하여 로그인 상태라면 해당 로그인된 계정의 아이디값을 통해 마이플레이스와 루틴 값을 모두 가져옴

  - 가져온 마이플레이스를 클릭 시 지도에 마커를 출력하고 출발지와 도착지를 설정할 수 있게 함

  - 가져온 루틴을 클릭 시 2 탭으로 넘어가 저장되어있는 경로를 지도에 표시해주고 출발지와 도착지의 검색창에 해당 좌표의 주소 값을 넣는다.

지도 출력

  내가 이번 프로젝트에서 가장 중요하다고 생각했던 것은 메인이 되는 신호등도 다른 기능도 아닌 지도라고 생각했고 일단 지도가 뜨지않으면 아무것도 할 수 없으며, 모든 작업이 지도 안에서 처리되는 과정이기에 가장먼저 지도 API를 알아봤다.

  지도 API를 알아보는 과정에서 카카오맵API 와 T-map API 둘중 어느것을 사용할지 팀원들과 의견을 주고받았고, 결과적으로 우리 프로그렘에 알맞다 생각해 T-Map API를 사용하기로 정했다
  지도를 띄우는 것 자체는 API가 잘되어있어 간단했는데, T-Map에 앱키를 요청하고 해당 앱키를 이용하여 T-map javascript를 호출하면 지도인 Tmap2.Map 를 호출할 수 있게된다.
  그 후 Tmap2.Map를 이용해 객체를 만들고, 해당 지도의 설정을 객체 안에서 해주면 해당 사진과 같이 지도가 출력된다.

<script script src="https://apis.openapi.sk.com/tmap/jsv2?version=1&appKey=발급받은 App key"></script>

위의 스크립트 문이 T-map javascript의 호출문이다

1탭: 지도 검색 기능

   지도검색 기능은 받아온 앱키로 T-map API에 ajax를 이용해 검색키워드, 검색기준 위치, 검색할 갯수를 양식에 맞춰 보내면 필요한 정보를 받아주는 방식으로 되어있다.
   나는 검색키워드는 input창으로 데이터를 받아올 수 있게 설정하고, 검색 위치는 지도인 map객체의 getCenter()를 이용해 현제 보고있는 지도의 위치를 기준으로 정보를 받아올 수 있게 했다.    API를 처음써본 나는 받아온 데이터객체에서 원하는 정보를 꺼내는데 어려움이 있었지만, 여러 시행착오 끝에 데이터객체 안의 원하는 데이터를 꺼낼 수 있게 되었고 해당 정보를 이용해 2탭인 경로탐색 기능과도 연계시켰다.

그리고 해당 기능을 1탭 HTML에 추가시키며 인터페이스를 적당히 만져줬다.
